# 公司內部點餐系統

純前端實作的公司內部點餐工具，使用 HTML5、CSS3、JavaScript，搭配 Google Identity Services 登入與 Google Sheets API 作為資料庫。

---

## 技術規格

| 項目 | 說明 |
|------|------|
| 前端框架 | 無（純 JavaScript） |
| 身分驗證 | Google Identity Services（GIS）Implicit Grant 流程 |
| 資料庫 | Google Sheets API v4（直接呼叫，無後端代理） |
| 相容環境 | 現代瀏覽器（Chrome / Edge / Firefox / Safari） |

---

## 檔案結構

```
├── index.html   # HTML 結構與外部腳本載入
├── all.css      # 樣式（行動優先，簡約現代風）
├── all.js       # 所有業務邏輯
└── .vscode/
    └── settings.json   # Live Server port 設定（5000）
```

## 初始設定

### 1. Google Cloud 專案

1. 前往 [Google Cloud Console](https://console.cloud.google.com/) 建立專案
2. 啟用 **Google Sheets API**
3. 前往「API 和服務 → 憑證」→ 建立 **OAuth 2.0 用戶端 ID**
   - 類型：**Web 應用程式**
   - 授權的 JavaScript 來源：填入實際使用的網址（例如 `http://127.0.0.1:5000`）
   - **不需要**填授權的重新導向 URI
4. 前往「OAuth 同意畫面」→ 將測試帳號加入**測試使用者**名單

### 2. 填入設定值（all.js 第 11–14 行）

```js
const CONFIG = {
    CLIENT_ID:      '你的 OAuth 用戶端 ID',
    SPREADSHEET_ID: '你的試算表 ID',
    ...
};
```

試算表 ID 位於 Google Sheets URL 的 `/d/` 與 `/edit` 之間。

### 3. 啟動方式

使用 VS Code **Live Server** 擴充功能開啟 `index.html`，預設 port 為 **5000**。

> ⚠️ 不可直接雙擊 HTML 檔（`file://` 協定不支援 Google OAuth）

## 功能說明

### 所有使用者

| 功能 | 說明 |
|------|------|
| Google 登入 | 使用 GIS Token Client 取得 Access Token |
| 身分驗證 | 登入後比對 Users 工作表的 Email 欄位 |
| 未授權提示 | Email 不在名單內時顯示錯誤畫面 |
| 今日點餐 | 依 TodayConfig 過濾 Menu，依餐廳與分類分組顯示 |
| 送出訂單 | 寫入 Orders 工作表（含時間、Email、餐廳、品名、金額、備註） |
| 今日訂單 | 列出當日所有訂單並顯示合計金額 |
| 一鍵複製 | 將訂單格式化為純文字，複製到剪貼簿（可貼入 LINE） |

### 管理員專屬

| 功能 | 說明 |
|------|------|
| 設定今日餐廳 | 從 Menu 撈出所有餐廳，勾選後寫入 TodayConfig |
| 清空今日點餐 | 刪除 Orders 工作表中除標題列外的所有資料 |

## 權限流程

```
登入
 └─ Email 在 Users 表？
      ├─ 否 → 顯示「未獲授權」畫面
      └─ 是 → 讀取權限欄位
               ├─ 管理員 → 顯示餐廳設定 + 點餐介面 + 訂單列表 + 清空按鈕
               └─ 一般成員 → 顯示點餐介面 + 訂單列表
```

## 注意事項

- 時間戳記以 `RAW` 模式寫入 Sheets，避免被自動轉為日期序號導致讀取異常
- 今日訂單以整數比對年/月/日，相容 Sheets 讀回時省略補零的情況（`4/18` vs `04/18`）
- Access Token 有效期約 1 小時，過期後需重新登入
- 本專案為純前端，所有 API 呼叫直接由瀏覽器發出，無後端伺服器
