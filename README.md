# 公司內部點餐系統

純前端實作的公司內部點餐工具，使用 HTML5、CSS3、JavaScript，搭配 Google Identity Services 登入與 Google Sheets API 作為資料庫，不需要任何後端伺服器。

---

## 技術規格

| 項目 | 說明 |
|---|---|
| 前端框架 | 無（純 JavaScript，無 Vue / React） |
| 身分驗證 | Google Identity Services（GIS）Implicit Grant 流程 |
| 資料庫 | Google Sheets API v4（瀏覽器直接呼叫，無後端代理） |
| 相容環境 | 現代瀏覽器（Chrome / Edge / Firefox / Safari） |

---

## 檔案結構

```
├── index.html            # HTML 結構與外部腳本載入
├── all.css               # 樣式（行動優先，簡約現代風）

├── README.md             # 本文件

```

## 初始設定

### 1. Google Cloud 專案

1. 前往 [Google Cloud Console](https://console.cloud.google.com/) 建立專案
2. 啟用 **Google Sheets API**
3. 前往「API 和服務 → 憑證」→ 建立 **OAuth 2.0 用戶端 ID**
   - 類型選擇：**Web 應用程式**
   - 授權的 JavaScript 來源：填入實際使用的網址（例如 `http://127.0.0.1:5000`）
   - 授權的重新導向 URI：**不需要填寫**
4. 前往「OAuth 同意畫面」→ 將使用者帳號加入**測試使用者**名單

### 2. 填入設定值

開啟 `all.js`，修改頂部的 CONFIG 區塊：

```js
const CONFIG = {
    CLIENT_ID:      '你的 OAuth 2.0 用戶端 ID',
    SPREADSHEET_ID: '你的 Google Sheets 試算表 ID',
};
```

> 試算表 ID 位於 Google Sheets 網址的 `/d/` 與 `/edit` 之間的那段字串。

### 3. 啟動方式

使用 VS Code **Live Server** 擴充功能在 `index.html` 上按右鍵 → Open with Live Server，預設 port 為 **5000**。

> ⚠️ 不可直接雙擊 HTML 檔開啟（`file://` 協定不支援 Google OAuth）

## Google Sheets 試算表結構

試算表共需 **5 張工作表**，名稱須與下方完全一致（含大小寫）：

---

### 工作表 1：`TodayConfig`

管理員設定當日開放點餐的餐廳，每列一筆。

| 今日開放餐廳 |
|---|
| 梁社漢排骨 |
| 50嵐 |

---

### 工作表 2：`Menu`

所有餐廳的完整菜單。`分類` 欄的值需與 `Options` 工作表的 `分類` 欄一致。

| 餐廳名稱 | 品名 | 單價 | 分類 |
|---|---|---|---|
| 梁社漢排骨 | 炸排骨飯 | 115 | 主食 |
| 梁社漢排骨 | 炸雞腿飯 | 130 | 主食 |
| 梁社漢排骨 | 秋葵 | 35 | 小菜 |
| 50嵐 | 珍珠奶茶(L) | 60 | 飲品 |

---

### 工作表 3：`Users`

授權使用者名單。`權限` 欄只有兩個合法值：`管理員` / `一般成員`。

| 姓名 | Email | 權限 |
|---|---|---|
| 管理員 01 | admin@example.com | 管理員 |
| 員工 01 | staff01@example.com | 一般成員 |
| 員工 02 | staff02@example.com | 一般成員 |

---

### 工作表 4：`Options`

附加選項設定，決定各分類的餐點在點餐時顯示哪些下拉選單。

| 分類 | 選項類型 | 選項值 | 預設值 |
|---|---|---|---|
| 主食 | 飯量 | 正常 | Y |
| 主食 | 飯量 | 飯多 | |
| 主食 | 飯量 | 飯少 | |
| 飲品 | 甜度 | 全糖 | |
| 飲品 | 甜度 | 七分糖 | |
| 飲品 | 甜度 | 半糖 | Y |
| 飲品 | 甜度 | 三分糖 | |
| 飲品 | 甜度 | 無糖 | |
| 飲品 | 冰量 | 正常冰 | Y |
| 飲品 | 冰量 | 少冰 | |
| 飲品 | 冰量 | 微冰 | |
| 飲品 | 冰量 | 去冰 | |

**欄位說明：**
- `分類`：需與 Menu 工作表的「分類」欄位完全一致
- `選項類型`：下拉選單的標題（例如「甜度」「冰量」）
- `選項值`：下拉選單的選項內容
- `預設值`：填 `Y` 代表該選項為頁面預設值，每個選項類型只設一個

> 若此工作表不存在或留空，點餐功能仍可正常運作（不顯示附加選項）

---

由系統自動寫入，**手動建立標題列**後不需要再操作。

| 點餐時間 | 訂購人 Email | 餐廳名稱 | 餐點內容 | 金額 | 備註 |
|---|---|---|---|---|---|
| 2026/04/19 12:05 | staff01@example.com | 梁社漢排骨 | 炸排骨飯（飯多） | 115 | 不要菜脯 |
| 2026/04/19 12:10 | staff02@example.com | 50嵐 | 珍珠奶茶(L)（半糖／少冰） | 60 | |

> 附加選項會自動附加在餐點內容後方，例如 `珍珠奶茶(L)（半糖／少冰）`

## 功能說明

### 所有已授權使用者

| 功能 | 說明 |
|---|---|
| Google 登入 | 使用 GIS Token Client 取得 Access Token |
| 身分驗證 | 登入後自動比對 Users 工作表的 Email 欄位 |
| 未授權提示 | Email 不在名單內時顯示錯誤畫面 |
| 今日點餐 | 依 TodayConfig 過濾 Menu，依餐廳與分類分組顯示菜單卡片 |
| 附加選項 | 依 Options 工作表，對應分類自動顯示飯量／甜度／冰量等下拉選單 |
| 送出訂單 | 將時間、Email、餐廳、餐點（含附加選項）、金額、備註寫入 Orders |
| 今日訂單 | 列出當日所有訂單並顯示合計金額 |
| 一鍵複製 | 將訂單格式化為純文字，複製到剪貼簿（適合貼入 LINE） |

### 管理員專屬

| 功能 | 說明 |
|---|---|
| 設定今日餐廳 | 從 Menu 撈出所有餐廳名稱，勾選後寫入 TodayConfig |
| 清空今日點餐 | 清除 Orders 工作表中除標題列外的所有訂單資料 |

## 權限流程

```
使用者登入
 └─ Email 在 Users 表？
      ├─ 否 → 顯示「未獲授權」畫面
      └─ 是 → 讀取權限欄位
               ├─ 管理員   → 餐廳設定 ＋ 點餐介面 ＋ 今日訂單 ＋ 清空按鈕
               └─ 一般成員 → 點餐介面 ＋ 今日訂單
```

---

## 注意事項

- 時間戳記以 `RAW` 模式寫入 Sheets，避免被自動辨識為日期序號而格式錯亂
- 今日訂單過濾以整數比對年／月／日，相容 Sheets 讀回時省略補零的情況（`4/19` vs `04/19`）
- Access Token 有效期約 1 小時，過期後需重新登入
- 所有 API 呼叫直接由瀏覽器發出，不需要後端伺服器與 secret key
